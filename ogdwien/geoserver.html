Fehlermeldungen

Geoserver unterstützt laut Dokumentation 3 verschiedene Service Exceptions. Wenn eine App einen Request an den Geoserver schickt muss dieser Request eines von den folgenden 3 Parametern enthalten um im Falle eines Fehlers das richtige Ausgabeformat zu bekommen.

  • XML: exceptions=text/xml - (default) XML output
  • JSON: exceptions=application/json - Simple JSON
  • JSONP: exceptions=text/javascript - Return a JsonP in the form: parseResponse(...jsonp...).

Wenn der Geoserver nicht verfügbar ist, wird momentan ein HTTP 404 an die Client-App geschickt.
Wenn der Response zu lange dauert (weil zu viele Daten übertragen werden), wird ein Timeout HTTP 408 retourniert.
